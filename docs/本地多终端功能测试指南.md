### 1.2 检查 ZooKeeper 状态

```bash
nc -zv 127.0.0.1 2181

# 如果 ZooKeeper 未运行，使用 Docker 启动
docker run -d --name zookeeper -p 2181:2181 zookeeper:3.5.9

# 创建根节点（首次连接时应用会自动创建，但也可以手动创建）
docker exec -it zookeeper zkCli.sh -server 127.0.0.1:2181
> create /libra-regions ""
> quit
```

## 2. 快速启动多客户端

### 2.1 启动命令模板

```bash
cd /Users/yuanyicheng/IdeaWorkspace/chat-room

# 客户端 1
npx tsx src/index.tsx \
  --port 9001 \
  --nickname "Alice" \
  --data-dir /tmp/chat-room/client1 \
  --log-dir /tmp/chat-room/client1/logs \
  --log-level debug

# 客户端 2
npx tsx src/index.tsx \
  --port 9002 \
  --nickname "Bob" \
  --data-dir /tmp/chat-room/client2 \
  --log-dir /tmp/chat-room/client2/logs \
  --log-level debug

# 客户端 3
npx tsx src/index.tsx \
  --port 9003 \
  --nickname "Charlie" \
  --data-dir /tmp/chat-room/client3 \
  --log-dir /tmp/chat-room/client3/logs \
  --log-level debug
```

### 2.2 命令行参数说明

| 参数 | 缩写 | 默认值 | 说明 |
|-----|------|--------|------|
| `--zk-addresses` | `-z` | `127.0.0.1:2181` | ZooKeeper 地址，多个用逗号分隔 |
| `--port` | `-p` | `9001` | P2P 监听端口，**多客户端必须不同** |
| `--config` | `-c` | `~/.chat-room/config.json` | 配置文件路径，**多客户端必须不同** |
| `--nickname` | `-n` | `User{随机数}` | 客户端昵称 |
| `--data-dir` | `-d` | `/tmp/chat-room` | 数据存储目录，**多客户端建议不同** |
| `--log-dir` | `-l` | `/tmp/chat-room/logs` | 日志输出目录，**多客户端建议不同** |
| `--log-level` | `-v` | `info` | 日志级别：debug/info/warn/error |
| `--help` | `-h` | - | 显示帮助信息 |

### 2.3 多客户端启动脚本

创建文件 `test-start.sh`：

```bash
#!/bin/bash
# test-start.sh - 启动多个聊天客户端进行测试

BASE_DIR="/tmp/chat-room"
LOG_LEVEL="debug"

# 创建目录
for i in 1 2 3; do
  mkdir -p "$BASE_DIR/client$i/logs"
done

# 启动客户端 1 - Alice
echo "Starting Alice on port 9001..."
npx tsx src/index.tsx \
  --port 9001 \
  --nickname "Alice" \
  --data-dir "$BASE_DIR/client1" \
  --log-dir "$BASE_DIR/client1/logs" \
  --log-level "$LOG_LEVEL" &

# 等待一小段时间让第一个客户端完全启动
sleep 2

# 启动客户端 2 - Bob
echo "Starting Bob on port 9002..."
npx tsx src/index.tsx \
  --port 9002 \
  --nickname "Bob" \
  --data-dir "$BASE_DIR/client2" \
  --log-dir "$BASE_DIR/client2/logs" \
  --log-level "$LOG_LEVEL" &

sleep 2

# 启动客户端 3 - Charlie
echo "Starting Charlie on port 9003..."
npx tsx src/index.tsx \
  --port 9003 \
  --nickname "Charlie" \
  --data-dir "$BASE_DIR/client3" \
  --log-dir "$BASE_DIR/client3/logs" \
  --log-level "$LOG_LEVEL" &

echo "All clients started!"
echo "Use 'killall node' to stop all clients"
```

```bash
chmod +x test-start.sh
./test-start.sh
```

## 3. 功能测试用例

### 3.1 配置与连接测试

#### TC-001: 首次启动配置流程

**测试步骤:**
1. 启动第一个客户端（使用空的配置目录）
2. 观察配置引导界面

**预期结果:**
- 显示 ZooKeeper 地址配置提示
- 可输入 ZooKeeper 地址（默认: 127.0.0.1:2181）
- 配置保存到 `~/.chat-room/config.json`

**验证方法:**
```bash
cat ~/.chat-room/config.json
```

#### TC-002: ZooKeeper 连接状态

**测试步骤:**
1. 启动客户端
2. 观察日志输出

**预期结果:**
```

[INFO] [App] ZooKeeper connected successfully
```

### 3.2 聊天室功能测试

#### TC-003: 创建并加入聊天室

**测试步骤:**
1. 客户端 1 输入聊天室 ID: `general`
2. 客户端 2 输入同一个聊天室 ID: `general`
3. 客户端 3 输入同一个聊天室 ID: `general`

**预期结果:**
- 所有客户端成功加入同一聊天室
- 成员列表显示所有在线用户
- 系统消息显示: `[时间] 系统: {昵称} 加入了聊天室`

#### TC-004: 成员上下线通知

**测试步骤:**
1. 启动 3 个客户端加入同一聊天室
2. 观察各客户端的成员列表和系统消息
3. 关闭客户端 2 (Bob)
4. 观察其他客户端的显示

**预期结果:**
- 客户端 2 关闭后，其他客户端显示: `[时间] 系统: Bob 离开了聊天室`
- 成员列表中 Bob 的状态变为灰色/离线标记

### 3.3 消息收发测试

#### TC-005: 发送和接收普通消息

**测试步骤:**
1. 客户端 1 (Alice) 输入: `Hello everyone!`
2. 客户端 2 (Bob) 输入: `Hi Alice!`
3. 客户端 3 (Charlie) 输入: `Good morning!`

**预期结果:**
- 所有消息格式: `[HH:mm:ss] 昵称: 消息内容`
- 消息实时同步到所有客户端

**测试步骤:**
1. 客户端 1 输入: `@Bob 你好`
2. 客户端 3 输入: `@Alice 和 @Charlie 你们好`

**预期结果:**
- 输入 `@` 后弹出在线成员列表供选择
- 被 @ 的消息高亮显示（黄色）
- 消息格式: `[HH:mm:ss] 昵称: @被@用户 消息内容`

#### TC-007: 回复消息功能

**测试步骤:**
1. 鼠标选中某条消息
2. 点击"回复"按钮
3. 输入回复内容

**预期结果:**
- 回复消息格式: `[HH:mm:ss] 昵称 [回复 原消息]: 回复内容`
- 显示引用的原始消息内容

## 5. 常见问题排查

### 5.1 端口被占用

**错误:**
```
Error: listen EADDRINUSE :::9001
```

**解决:**
```bash
# 查找占用端口的进程
lsof -i :9001

# 使用其他端口启动
npx tsx src/index.tsx --port 9004
```

### 5.2 ZooKeeper 连接失败

**错误:**
```
Failed to connect to ZooKeeper
```

**解决:**
1. 检查 ZooKeeper 是否运行: `docker ps | grep zookeeper`
2. 如果未运行: `docker start zookeeper`
3. 检查端口: `nc -zv 127.0.0.1 2181`

### 5.3 配置文件冲突

**问题:** 多客户端使用相同配置目录导致覆盖

**解决:** 每个客户端使用独立的 `--config` 和 `--data-dir`

```bash
# 客户端 1
--config ~/.chat-room/client1/config.json --data-dir /tmp/chat-room/client1

# 客户端 2
--config ~/.chat-room/client2/config.json --data-dir /tmp/chat-room/client2
```

### 5.4 日志级别选择

| 级别 | 适用场景 |
|-----|---------|
| `debug` | 详细调试信息，网络包解析 |
| `info` | 正常运行，默认级别 |
| `warn` | 只显示警告和错误 |
| `error` | 只显示错误 |
