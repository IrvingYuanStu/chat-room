## 1. 环境准备

### 1.1 系统要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| macOS | 12.0 (Monterey) | 14.0 (Sonoma) 或更高 |
| Node.js | 22.0.0 | 22.x LTS |
| npm | 9.0.0 | 10.x |
| ZooKeeper | 3.5.9 | 3.8.x 或 3.9.x |
| 内存 | 4GB | 8GB 或更高 |
| 磁盘空间 | 500MB | 1GB 或更高 |

### 1.2 安装 Node.js

```bash
# 使用 nvm 安装（推荐）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 22
nvm use 22

# 验证安装
node --version  # 应显示 v22.x.x
npm --version   # 应显示 10.x.x
```

### 1.3 安装 ZooKeeper

#### 方式一：使用 Homebrew（推荐）

```bash
# 安装 ZooKeeper
brew install zookeeper

# 启动 ZooKeeper
zkServer start

# 验证状态
zkServer status

# 查看日志
tail -f /usr/local/var/log/zookeeper/zookeeper.log
```

#### 方式二：使用 Docker

```bash
# 拉取 ZooKeeper 镜像
docker pull zookeeper:3.5.9

# 启动单节点 ZooKeeper
docker run --name zookeeper \
  -p 2181:2181 \
  -e ZOO_MY_ID=1 \
  -e ZOO_SERVERS="server.1=0.0.0.0:2888:3888;2181" \
  -d zookeeper:3.5.9

# 查看日志
docker logs -f zookeeper

# 停止 ZooKeeper
docker stop zookeeper
docker rm zookeeper
```

#### 方式三：使用 Docker Compose（集群模式）

```bash
# 创建 docker-compose.yml
cat > docker-compose-zk.yml <<EOF
version: '3'
services:
  zoo1:
    image: zookeeper:3.5.9
    container_name: zoo1
    ports:
      - "2181:2181"
    environment:
      ZOO_MY_ID: 1
      ZOO_SERVERS: server.1=0.0.0.0:2888:3888;2181

  zoo2:
    image: zookeeper:3.5.9
    container_name: zoo2
    ports:
      - "2182:2181"
    environment:
      ZOO_MY_ID: 2
      ZOO_SERVERS: server.1=zoo1:2888:3888;2181 server.2=0.0.0.0:2888:3888;2181

  zoo3:
    image: zookeeper:3.5.9
    container_name: zoo3
    ports:
      - "2183:2181"
    environment:
      ZOO_MY_ID: 3
      ZOO_SERVERS: server.1=zoo1:2888:3888;2181 server.2=zoo2:2888:3888;2181 server.3=0.0.0.0:2888:3888;2181
EOF

# 启动集群
docker-compose -f docker-compose-zk.yml up -d

# 验证集群状态
echo ruok | nc localhost 2181
echo ruok | nc localhost 2182
echo ruok | nc localhost 2183
# 应返回 imok
```

### 1.4 配置 ZooKeeper

编辑 ZooKeeper 配置文件：

```bash
# Homebrew 安装路径
vi /usr/local/etc/zookeeper/zoo.cfg

# Docker 方式通过环境变量配置
```

关键配置项：

```properties
# 心跳间隔（毫秒）
tickTime=2000

# 初始化同步超时（tickTime 的倍数）
initLimit=10

# 同步超时（tickTime 的倍数）
syncLimit=5

# 数据目录
dataDir=/usr/local/var/lib/zookeeper

# 客户端连接端口
clientPort=2181

# 最大客户端连接数
maxClientCnxns=60

# 会话超时（毫秒）
maxSessionTimeout=60000
minSessionTimeout=4000
```

### 4.3 多机部署（生产环境）

#### 步骤 1：准备环境

每台机器执行：

```bash
# 1. 安装 Node.js 22
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 22

# 2. 克隆代码
git clone <repository-url> /opt/chat-room
cd /opt/chat-room

# 3. 安装依赖
npm install --production

# 4. 构建
npm run build
```

#### 步骤 2：配置 ZooKeeper 集群

在 ZooKeeper 服务器上部署集群（参考 1.3 节）。

#### 步骤 3：创建启动脚本

创建 `/opt/chat-room/start.sh`：

```bash
#!/bin/bash
# Chat Room 启动脚本

APP_DIR="/opt/chat-room"
LOG_DIR="/var/log/chat-room"
PID_FILE="/var/run/chat-room.pid"

# 创建日志目录
mkdir -p "$LOG_DIR"

# 启动应用
cd "$APP_DIR"
nohup node dist/index.js >> "$LOG_DIR/app.log" 2>&1 &
echo $! > "$PID_FILE"

echo "Chat Room started with PID: $(cat $PID_FILE)"
```

#### 步骤 4：创建停止脚本

创建 `/opt/chat-room/stop.sh`：

```bash
#!/bin/bash
# Chat Room 停止脚本

PID_FILE="/var/run/chat-room.pid"

if [ -f "$PID_FILE" ]; then
  PID=$(cat "$PID_FILE")
  kill -TERM "$PID"
  rm -f "$PID_FILE"
  echo "Chat Room stopped (PID: $PID)"
else
  echo "Chat Room is not running"
fi
```

#### 步骤 5：配置 systemd 服务（可选）

创建 `/etc/systemd/system/chat-room.service`：

```ini
[Unit]
Description=Chat Room Terminal Tool
After=network.target zookeeper.service

[Service]
Type=simple
User=chatroom
Group=chatroom
WorkingDirectory=/opt/chat-room
ExecStart=/usr/bin/node /opt/chat-room/dist/index.js
Restart=on-failure
RestartSec=10

# 日志
StandardOutput=journal
StandardError=journal
SyslogIdentifier=chat-room

# 安全
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
# 创建专用用户
sudo useradd -r -s /bin/false chatroom
sudo chown -R chatroom:chatroom /opt/chat-room

# 重载 systemd
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable chat-room

# 启动服务
sudo systemctl start chat-room

# 查看状态
sudo systemctl status chat-room

# 查看日志
sudo journalctl -u chat-room -f
```

### 5.4 创建安装器

创建 `scripts/install.sh`：

```bash
#!/bin/bash
# Chat Room 安装脚本

VERSION="0.1.0"
INSTALL_DIR="/opt/chat-room"
BIN_LINK="/usr/local/bin/chat-room"

# 检查 Node.js
if ! command -v node &> /dev/null; then
  echo "错误: 未安装 Node.js"
  echo "请访问 https://nodejs.org/ 下载安装"
  exit 1
fi

# 检查 Node 版本
NODE_VERSION=$(node -v | cut -d'v' -f2 | cut -d'.' -f1)
if [ "$NODE_VERSION" -lt 22 ]; then
  echo "错误: Node.js 版本需要 >= 22.0.0"
  echo "当前版本: $(node -v)"
  exit 1
fi

# 创建安装目录
sudo mkdir -p "$INSTALL_DIR"

# 复制文件
sudo cp -r . "$INSTALL_DIR"

# 安装依赖
cd "$INSTALL_DIR"
npm install --production

# 创建符号链接
sudo ln -sf "$INSTALL_DIR/dist/index.js" "$BIN_LINK"

# 设置权限
sudo chmod +x "$BIN_LINK"

echo "✓ Chat Room 安装成功！"
echo "  运行 'chat-room' 启动程序"
```

## 6. 故障排查

### 6.1 常见问题

#### 问题 1：ZooKeeper 连接失败

**现象**：
```
加载聊天室列表失败: Error: connect ENOTFOUND 127.0.0.1:2181
```

**解决方案**：
```bash
# 检查 ZooKeeper 是否运行
zkServer status

# 检查端口是否监听
netstat -an | grep 2181

# 检查防火墙
sudo ufw status

# 查看 ZooKeeper 日志
tail -f /usr/local/var/log/zookeeper/zookeeper.log
```

#### 问题 2：P2P 连接建立失败

**现象**：
```
Failed to connect to peer 张三 at 192.168.1.100:9001
```

**解决方案**：
```bash
# 检查目标机器是否在线
ping 192.168.1.100

# 检查端口是否开放
nc -zv 192.168.1.100 9001

# 检查本地防火墙
sudo ufw status

# 查看应用日志
tail -f /var/log/chat-room/app.log
```

#### 问题 3：消息未送达

**现象**：发送消息后对方未收到

**排查步骤**：
```bash
# 1. 检查双方是否在同一聊天室
# 查看 ~/.chat-room/config.json 中的 currentRoomId

# 2. 检查 P2P 连接状态
# 查看应用日志中的 "peer-connected" 事件

# 3. 检查心跳超时
# 日志中会有 "Peer xxx timeout" 提示

# 4. 测试网络连通性
ping <对方IP>
telnet <对方IP> 9001
```

#### 问题 4：内存占用过高

**现象**：进程内存占用超过预期

**解决方案**：
```bash
# 查看内存使用
ps aux | grep "node.*chat-room"

# 限制消息缓存大小（代码中已有 500 条限制）
# 清理历史消息
# rm -rf /tmp/chat-room/*/

# 重启应用
chat-room
```
