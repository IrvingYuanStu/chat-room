# 开发者构建指南

## 构建 Node.js 二进制包

本指南说明如何将 Chat Room Node.js 应用打包成独立的可执行文件。

## 工具说明

我们使用 **pkg** 来打包 Node.js 应用，它可以将 Node.js 代码编译成独立的二进制可执行文件。

### pkg 工作原理

1. 将 Node.js 应用及其依赖打包
2. 包含 Node.js 运行时
3. 生成可直接执行的二进制文件
4. 用户无需安装 Node.js 环境

## 快速开始

### 1. 安装依赖

```bash
npm install
```

项目已包含 `pkg` 作为开发依赖。

### 2. 构建二进制包

```bash
# 使用构建脚本（推荐）
./scripts/build-binaries.sh

# 或使用 npm 命令
npm run package

# 或为特定平台构建
npm run package:macos   # 仅 macOS
npm run package:linux   # 仅 Linux
npm run package:win     # 仅 Windows
```

### 3. 查看输出

构建完成后，二进制文件位于 `binaries/` 目录：

```
binaries/
├── chat-room-macos          # macOS 可执行文件
├── chat-room-linux          # Linux 可执行文件
├── chat-room-win.exe        # Windows 可执行文件
├── chat-room-macos-x64.tar.gz
├── chat-room-linux-x64.tar.gz
└── chat-room-win-x64.zip
```

## 配置说明

### pkg.json 配置文件

```json
{
  "name": "chat-room",
  "version": "0.1.0",
  "bin": "dist/index.js",
  "outputPath": "binaries",
  "targets": [
    "node18-macos-x64",
    "node18-linux-x64",
    "node18-win-x64"
  ],
  "outputPattern": "<name>-<version>-<platform>-<arch>"
}
```

**配置项说明：**
- `bin`: 入口文件（必须先编译 TypeScript）
- `outputPath`: 输出目录
- `targets`: 目标平台和 Node.js 版本
- `assets`: 需要包含的非 JS 文件

### package.json 脚本

```json
{
  "scripts": {
    "package": "npm run build && pkg .",
    "package:macos": "npm run build && pkg . --targets node18-macos-x64 --output ./binaries/chat-room-macos",
    "package:linux": "npm run build && pkg . --targets node18-linux-x64 --output ./binaries/chat-room-linux",
    "package:win": "npm run build && pkg . --targets node18-win-x64 --output ./binaries/chat-room-win.exe"
  }
}
```

## 构建脚本详解

### scripts/build-binaries.sh

这个自动化脚本执行以下步骤：

1. **清理旧的构建**
   ```bash
   rm -rf binaries
   mkdir -p binaries
   ```

2. **编译 TypeScript**
   ```bash
   npm run build
   ```

3. **打包各平台二进制**
   ```bash
   pkg . --targets node18-macos-x64 --output ./binaries/chat-room-macos
   pkg . --targets node18-linux-x64 --output ./binaries/chat-room-linux
   pkg . --targets node18-win-x64 --output ./binaries/chat-room-win.exe
   ```

4. **创建压缩包**
   ```bash
   tar -czf chat-room-macos-x64.tar.gz chat-room-macos
   tar -czf chat-room-linux-x64.tar.gz chat-room-linux
   zip chat-room-win-x64.zip chat-room-win.exe
   ```

5. **显示构建结果**
   ```
   📊 Build Summary:
   chat-room-macos          55.2 MB
   chat-room-linux          54.8 MB
   chat-room-win.exe        56.1 MB
   ```

## 优化二进制大小

### 当前大小

- 未压缩：约 50-60 MB
- 压缩后：约 15-20 MB

### 减小体积的方法

#### 1. 使用 UPX 压缩

```bash
# 安装 UPX
brew install upx  # macOS
sudo apt install upx  # Linux

# 压缩二进制文件
upx --best --lzma binaries/chat-room-macos
upx --best --lzma binaries/chat-room-linux

# 压缩后可减小到 15-20 MB
```

#### 2. 排除不必要的依赖

在 `pkg.json` 中添加：

```json
{
  "scripts": [
    "dist/**/*.js"
  ],
  "assets": [],
  "exclude": [
    "node_modules/not-needed/**"
  ]
}
```

#### 3. 使用不同的 Node.js 版本

```bash
# 使用更小的 Node.js 运行时
pkg . --targets node16-linux-x64  # 比 node18 小
```

## 常见问题

### Q: 为什么二进制文件这么大？

A: pkg 将整个 Node.js 运行时打包进去了。这是正常的。可以通过 UPX 压缩减小体积。

### Q: 如何支持 ARM 架构（如 Apple Silicon）？

A: 在 `pkg.json` 中添加目标：

```json
{
  "targets": [
    "node18-macos-arm64",
    "node18-linux-arm64"
  ]
}
```

### Q: 二进制文件运行时报错找不到模块

A: 确保该模块在 `package.json` 的 `dependencies` 中（不是 `devDependencies`）。pkg 只打包 `dependencies`。

### Q: 如何调试二进制文件？

A: pkg 支持源码映射：

```bash
pkg . --public --public-packaging
```

或者在打包前添加日志语句。

### Q: Windows 杀毒软件报警怎么办？

A: 这是误报，因为二进制文件包含 Node.js 运行时。可以：
1. 在发布前签名代码
2. 用户添加到杀毒软件白名单
3. 发布到可信平台（如 GitHub）

## 持续集成（CI/CD）

### GitHub Actions 示例

创建 `.github/workflows/release.yml`:

```yaml
name: Build and Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        include:
          - os: ubuntu-latest
            target: node18-linux-x64
            output: chat-room-linux
          - os: macos-latest
            target: node18-macos-x64
            output: chat-room-macos
          - os: windows-latest
            target: node18-win-x64
            output: chat-room-win.exe

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build

      - name: Package binary
        run: npx pkg . --targets ${{ matrix.target }} --output ${{ matrix.output }}

      - name: Upload to Release
        uses: softprops/action-gh-release@v1
        with:
          files: ${{ matrix.output }}
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## 安全考虑

### 1. 代码签名

**macOS:**
```bash
# 需要 Apple Developer 证书
codesign --deep --force --verify --verbose --sign "Developer ID Application: Your Name" chat-room-macos
```

**Windows:**
```bash
# 需要代码签名证书
signtool sign /f certificate.pfx /p password chat-room-win.exe
```

### 2. 校验和

发布时提供 SHA256 校验和：

```bash
shasum -a 256 binaries/* > binaries/checksums.txt
```

### 3. 安全扫描

```bash
npm audit
npm audit fix

# 或使用 Snyk
npm install -g snyk
snyk test
```
