# 二进制打包限制说明

## 当前状况

由于 Chat Room 项目使用 ES Modules（`type: "module"`），而 `pkg` 工具对 ES modules 的支持有限，目前**无法直接打包成二进制文件**。

## 技术原因

### 1. ES Modules 兼容性

项目使用：
- ES modules imports (`import`/`export`)
- Top-level await
- 动态 import
- React (Ink) UI 框架

`pkg` 工具目前：
- 主要支持 CommonJS 格式
- 对 ES modules 支持有限
- 打包大型 React 应用时会遇到路径问题

### 2. 依赖问题

Ink（终端UI框架）及其依赖：
```javascript
// ink 内部使用
await import('./devtools.js');  // top-level await
import devtools from 'react-devtools-core';  // pkg无法解析
```

## 推荐的部署方案

### ✅ 方案 1: 源码安装（推荐）

**优点：**
- 完全支持所有特性
- 配置灵活
- 易于调试

**缺点：**
- 用户需要安装 Node.js

```bash
curl -fsSL https://raw.githubusercontent.com/yourusername/chat-room/main/install.sh | bash
```

### ✅ 方案 2: Docker 部署

**优点：**
- 开箱即用
- 环境隔离
- 易于管理

**缺点：**
- 需要 Docker

```bash
git clone https://github.com/yourusername/chat-room.git
cd chat-room
docker-compose up -d
```

### ⚠️ 方案 3: NPM 全局安装

**优点：**
- 简单快速

**缺点：**
- 需要安装 Node.js

```bash
npm install -g chat-room
chat-room
```

## 未来二进制支持方案

### 选项 A: 等待 pkg 完善 ES modules 支持

关注 [pkg issues](https://github.com/vercel/pkg/issues) 中 ES modules 相关问题的进展。

### 选项 B: 使用其他打包工具

#### nexe

```bash
npm install -g nexe
nexe dist/index.js -o chat-room-macos
```

限制：同样对 ES modules 支持有限。

#### Node.js 封装工具

创建独立的 Node.js 运行时包：

```bash
# 复制 Node.js 运行时
cp -r /usr/local/bin/node runtime/
cp -r /usr/local/lib/node_modules runtime/

# 添加应用代码
cp -r dist runtime/app/

# 创建启动脚本
cat > runtime/chat-room << 'EOF'
#!/bin/bash
dirname=$(dirname "$0")
"$dirname/node" "$dirname/app/index.js"
EOF
```

### 选项 C: 改造项目为 CommonJS

将项目从 ES modules 改为 CommonJS：

1. 移除 `package.json` 中的 `"type": "module"`
2. 将所有 `.js` 改为 `.cjs`
3. 将 `import` 改为 `require`
4. 处理动态 import

**工作量：** 大
**影响：** 可能引入新问题

## 当前建议

对于最终用户，推荐以下方式：

### 个人用户
- 使用一键安装脚本（需要 Node.js）
- 或使用 Docker

### 企业/团队
- Docker 部署（推荐）
- 或在团队内维护私有 npm 镜像

### 开发者
- 源码安装
- 或本地构建

## 示例：创建独立的 Node.js 包

创建一个包含 Node.js 运行时的独立包：

```bash
#!/bin/bash
# 创建独立应用包脚本

APP_NAME="chat-room"
VERSION=$(node -p "require('./package.json').version")
PKG_DIR="$APP_NAME-$VERSION-osx-x64"

mkdir -p "$PKG_DIR"

# 复制 Node.js 运行时（假设系统已安装）
NODE_VERSION=$(node -v | cut -d'v' -f2)
cp /usr/local/bin/node "$PKG_DIR/"

# 复制应用代码
mkdir -p "$PKG_DIR/app"
cp -r dist "$PKG_DIR/app/"
cp package.json "$PKG_DIR/app/"

# 复制依赖
mkdir -p "$PKG_DIR/node_modules"
cp -r node_modules/* "$PKG_DIR/node_modules/"

# 创建启动脚本
cat > "$PKG_DIR/chat-room" << 'EOF'
#!/bin/bash
SCRIPT_DIR=$(dirname "$0")
export NODE_PATH="$SCRIPT_DIR/node_modules"
"$SCRIPT_DIR/node" "$SCRIPT_DIR/app/dist/index.js"
EOF

chmod +x "$PKG_DIR/chat-room"

# 打包
tar -czf "$PKG_DIR.tar.gz" "$PKG_DIR"

echo "Package created: $PKG_DIR.tar.gz"
```

## 总结

| 方案 | 难度 | 推荐度 | 说明 |
|------|------|--------|------|
| 一键安装脚本 | ⭐ | ✅ 推荐 | 需要用户安装 Node.js |
| Docker | ⭐⭐ | ✅ 推荐 | 开箱即用 |
| NPM 全局安装 | ⭐ | ✅ 推荐 | 简单快速 |
| pkg 二进制 | ⭐⭐⭐⭐ | ❌ 暂不支持 | ES modules 兼容性问题 |
| Node.js 独立包 | ⭐⭐⭐ | ⚠️ 可选 | 包体积较大 |
