## 1. 快速入门

### 1.1 什么是 Chat Room？

Chat Room 是一个运行在 macOS 终端内的局域网聊天工具，具有以下特点：

- ✅ **无需登录注册** - 启动即可使用
- ✅ **自动发现成员** - 加入聊天室即可看到所有在线人员
- ✅ **纯文本界面** - 轻量快速，适合终端用户
- ✅ **消息本地保存** - 重启后可查看历史记录

### 1.2 首次使用

#### 步骤 1：启动程序

```bash
chat-room
```

或

```bash
npm start
```

#### 步骤 2：配置连接

首次启动时会提示配置：

```
配置 ZooKeeper 地址
请输入 ZooKeeper 集群地址，格式: host:port
多个地址用逗号分隔，例如: 127.0.0.1:2181,127.0.0.1:2182
> 127.0.0.1:2181
```

**提示**：
- 单个 ZK 节点：`127.0.0.1:2181`
- 多个 ZK 节点（集群）：`127.0.0.1:2181,127.0.0.1:2182,127.0.0.1:2183`
- 默认端口：`2181`

#### 步骤 3：设置昵称

```
配置昵称
请输入您的显示昵称 (1-20 个字符)
> 张三
```

**昵称规则**：
- 长度：1-20 个字符
- 支持中文、字母、数字
- 支持下划线 `_` 和连字符 `-`
- 不能包含特殊符号

#### 步骤 4：选择聊天室

```
选择聊天室
1. general (3 位成员)
2. dev-team (5 位成员)
3. random (2 位成员)

输入聊天室 ID 创建新聊天室，或按序号选择
> 1
```

**选项**：
- 输入序号（1, 2, 3...）加入现有聊天室
- 输入新的聊天室 ID 自动创建并加入

## 2. 界面介绍

### 2.1 顶部状态栏

```
chat-room - [general]        ● 已连接    在线: 5
   ↑                    ↑        ↑          ↑
  应用名              聊天室   连接状态    成员数
```

| 组件 | 说明 |
|------|------|
| `chat-room` | 应用名称 |
| `[general]` | 当前聊天室 ID |
| `● 已连接` | ZooKeeper 连接状态（绿色=已连接，黄色=断开中） |
| `在线: 5` | 当前聊天室在线成员数量 |

### 2.2 聊天消息区域（左侧 70%）

显示聊天消息历史，最新消息在最下方。

**消息类型**：

| 类型 | 格式 | 示例 |
|------|------|------|
| 普通消息 | `[时间] 昵称: 内容` | `[14:30:28] 张三: 大家好` |
| 自己的消息 | `[时间] 我: 内容` | `[14:30:30] 我: 你好` |
| 系统消息 | `[时间] 系统: 内容`（灰色） | `[14:30:25] 系统: 张三 加入了聊天室` |
| @ 消息 | `[时间] 昵称: @某人 内容`（黄色高亮） | `[14:30:35] 李四: @张三 欢迎！` |
| 回复消息 | `[时间] 昵称 [回复 原消息]: 内容` | `[14:30:40] 张三 [回复 李四]: 谢谢！` |

### 2.3 成员列表区域（右侧 30%）

显示聊天室所有成员。

| 图标 | 含义 |
|------|------|
| ● 名称 | 在线成员（绿色） |
| ○ 名称 | 离线成员（灰色） |

成员按上线时间排序，最新加入的在最上面。

### 2.4 输入区域（左下角）

```
[张三] ___________________
  ↑           ↑
 昵称      输入框
```

## 3. 基本操作

### 3.1 发送消息

#### 发送单行消息

1. 在输入框中输入内容
2. 按 `Enter` 发送

```
[张三] 大家好！_____________
[按 Enter]
```

#### 发送多行消息

1. 输入第一行内容
2. 按 `Shift + Enter` 换行
3. 继续输入下一行
4. 按 `Enter` 发送

```
[张三] 第一行_____________
[按 Shift+Enter]
[张三] 第一行
     第二行_____________
[按 Enter 发送]
```

### 3.2 查看历史消息

历史消息会自动保存在本地，重启程序后会自动加载。

- **最多显示**：最近 500 条消息
- **保存位置**：`/tmp/chat-room/{聊天室ID}/history.txt`
- **自动清理**：超过 30 天的消息会在启动时自动清理

### 3.3 退出聊天室

**方式一：使用命令**

```
[张三] /exit-room_____________
```

系统会提示确认：

```
确定退出聊天室 general? (y/n)
```

- 输入 `y` 确认退出
- 输入 `n` 取消

**方式二：使用快捷键**

按 `Ctrl+C` 会直接退出整个程序，不推荐用于仅退出聊天室。

### 3.4 退出程序

**方式一：使用命令**

```
[张三] /quit_____________
```

**方式二：使用快捷键**

按 `Ctrl+C` 强制退出。

## 4. 高级功能

### 4.1 @ 提醒功能

#### 提醒特定成员

1. 输入 `@` 符号
2. 系统自动弹出成员列表
3. 继续输入昵称进行过滤
4. 选择成员后按 `Tab` 或 `Enter` 确认

**示例**：

```
[张三] @张_____________
       ↑
   自动弹出成员列表：
     @张小明
     @张三
   > @张三刚

[按 Tab 确认]

[张三] @张三刚 你好_____________
```

**提示**：
- 被 @ 的消息会以**黄色高亮**显示
- 目标成员会更容易注意到消息
- 如果 @ 了离线成员，会收到警告提示

### 4.2 回复消息

#### 回复特定消息

1. 使用 `↑` / `↓` 键浏览历史消息
2. 选中要回复的消息（按 `↑` 键）
3. 按 `Enter` 进入回复模式
4. 输入回复内容
5. 按 `Enter` 发送

**示例**：

```
步骤 1：按 ↑ 键选中消息
[14:30:35] 李四: 欢迎大家加入聊天室！
            ↑ 选中高亮 [回复 (Enter)]

步骤 2：按 Enter 进入回复模式
[张三] > 李四: 欢迎大家加入聊天室！
     _______________________

步骤 3：输入回复内容
[张三] > 李四: 欢迎大家加入聊天室！
     谢谢李四！____________

步骤 4：按 Enter 发送
[14:30:40] 张三 [回复 李四]: 谢谢李四！
```

**提示**：
- 按 `Esc` 可退出回复模式
- 回复消息会显示被回复的消息内容（灰色引用）

### 4.3 修改昵称

使用 `/rename` 命令修改昵称：

```
[张三] /rename 张小三__________
```

修改成功后：

```
[14:31:00] 系统: 张三 修改昵称为 张小三
```

**注意事项**：
- 新昵称会立即在成员列表中更新
- 历史消息中的昵称保持不变
- 聊天室内所有成员都会收到昵称修改通知

### 4.4 多聊天室切换

程序支持同时加入多个聊天室（虽然同一时间只能在一个聊天室内）。

#### 切换聊天室

1. 使用 `/exit-room` 退出当前聊天室
2. 在聊天室选择界面选择另一个聊天室

#### 最近访问的聊天室

聊天室选择界面会显示最近访问的聊天室，并标记为 `[最近]`：

```
选择聊天室:
  1. general (3 位成员) [最近]
  2. dev-team (5 位成员)
```

### 4.5 网络状态监控

#### 连接状态

顶部状态栏显示 ZooKeeper 连接状态：

| 状态 | 显示 | 说明 |
|------|------|------|
| 已连接 | `● 已连接`（绿色） | 正常连接到 ZooKeeper |
| 断开中 | `● 断开中`（黄色） | 连接断开，正在重连 |
| 已重连 | `● 已重连`（绿色） | 重连成功（显示 3 秒后恢复） |

#### 连接断开处理

如果 ZooKeeper 连接断开：

1. 系统会自动尝试重连（每 5 秒一次）
2. 重连期间仍可查看历史消息
3. 已建立的 P2P 连接仍然可用
4. 重连成功后会显示提示

## 6. 常见问题

### 6.1 安装与启动

#### Q1: 提示 "command not found: chat-room"

**原因**：未安装或未正确配置环境变量

**解决方案**：

```bash
# 方式一：使用 npm start
npm start

# 方式二：全局安装
npm link
chat-room

# 方式三：直接运行
node /path/to/chat-room/dist/index.js
```

#### Q2: 提示 "Cannot connect to ZooKeeper"

**原因**：ZooKeeper 服务未启动或地址配置错误

**解决方案**：

```bash
# 检查 ZooKeeper 是否运行
zkServer status

# 如果未运行，启动它
zkServer start

# 或使用 Docker
docker start zookeeper
```

#### Q3: Node.js 版本不兼容

**原因**：Node.js 版本低于 22.0.0

**解决方案**：

```bash
# 检查当前版本
node --version

# 使用 nvm 安装正确版本
nvm install 22
nvm use 22
```

### 6.2 聊天功能

#### Q4: 发送消息后对方收不到

**可能原因**：

1. **对方已离线** - 检查成员列表中对方的状态（绿色 ● 为在线）
2. **网络问题** - 检查网络连接，ping 对方 IP
3. **P2P 连接未建立** - 查看应用日志（开发模式）
4. **在不同聊天室** - 确认双方在同一聊天室

**排查步骤**：

```bash
# 1. 检查对方是否在线
# 查看成员列表，对方是否为绿色 ●

# 2. 检查网络连通性
ping <对方IP>

# 3. 检查端口
telnet <对方IP> 9001

# 4. 重启程序
chat-room
```

#### Q5: 被 @ 的消息没有高亮

**原因**：昵称输入不完整

**解决方案**：

- 使用 `@` 符号后等待成员列表弹出
- 按 `Tab` 或 `Enter` 确认选择，不要手动输入完整昵称

#### Q6: 历史消息丢失

**原因**：历史文件被删除或清理

**解决方案**：

- 历史消息保存在 `/tmp/chat-room/{聊天室ID}/history.txt`
- `/tmp` 目录在系统重启后会被清空
- 如需持久保存，建议定期备份历史文件

```bash
# 备份历史消息
cp -r /tmp/chat-room ~/chat-room-backup
```

### 6.3 性能与稳定

#### Q7: 程序占用内存过高

**正常范围**：< 100MB

**解决方案**：

```bash
# 查看进程内存
ps aux | grep chat-room

# 如果内存过高，重启程序
# 限制消息数量（自动限制为 500 条）
```

#### Q8: 终端显示异常

**现象**：界面乱码、显示不完整

**解决方案**：

```bash
# 调整终端窗口大小
# 确保终端至少 80x24 字符

# 重置终端
reset

# 重启程序
chat-room
```

#### Q9: 快捷键不响应

**可能原因**：

1. 终端快捷键冲突
2. 输入法问题
3. 程序未正确捕获键盘事件

**解决方案**：

- 切换到英文输入法
- 确保终端窗口获得焦点
- 尝试使用命令代替快捷键

### 6.4 高级问题

#### Q10: 如何查看详细日志？


## 7. 最佳实践

### 7.1 使用技巧

#### 1. 合理设置昵称

**推荐**：
- 使用真实姓名或常用昵称
- 容易识别（如 "张三-开发组"）
- 避免频繁修改

**不推荐**：
- 使用特殊字符（如 `😀张三`）
- 过长或过短的昵称
- 容易混淆的昵称（如多个 "用户1"）

#### 2. 聊天室管理

**创建聊天室**：
- 使用有意义的聊天室 ID（如 `dev-team`, `general`）
- 不同团队使用不同聊天室
- 定期清理不活跃的聊天室

**加入聊天室**：
- 确认聊天室用途再加入
- 遵守聊天室规则
- 及时退出不再使用的聊天室

#### 3. 消息发送

**高效沟通**：
- 使用 @ 提及相关人员
- 使用回复功能保持上下文
- 避免刷屏（短时间内发送大量消息）

**消息格式**：
```
推荐：
[14:30:00] 张三: @李四 请问 Bug 修复了吗？

不推荐：
[14:30:00] 张三: @李四
[14:30:01] 张三: 请问
[14:30:02] 张三: Bug
[14:30:03] 张三: 修复了吗？
```

### 7.2 网络配置

#### 局域网环境

**最佳配置**：
- 使用有线网络（更稳定）
- 确保所有设备在同一子网
- 关闭不必要的防火墙规则

**检查连通性**：

```bash
# 测试网络
ping <对方IP>

# 测试端口
nc -zv <对方IP> 2181  # ZooKeeper
nc -zv <对方IP> 9001  # P2P
```

#### 跨子网访问

如果需要跨子网访问：

1. 确保路由器支持跨子网通信
2. 配置 ZooKeeper 集群地址为外网可访问地址
3. 开放防火墙端口（2181, 9001-9010）

### 7.3 数据管理

#### 历史消息备份

**定期备份**：

```bash
# 创建备份脚本
cat > backup-chat.sh <<'EOF'
#!/bin/bash
BACKUP_DIR=~/chat-room-backup
DATE=$(date +%Y%m%d)
mkdir -p "$BACKUP_DIR"
cp -r /tmp/chat-room "$BACKUP_DIR/$DATE"
echo "Backup created: $BACKUP_DIR/$DATE"
EOF

chmod +x backup-chat.sh

# 每周备份（添加到 crontab）
crontab -e
# 添加：0 0 * * 0 /path/to/backup-chat.sh
```

#### 配置文件管理

**多机器配置**：

如果需要在多台机器上使用，可以复制配置文件：

```bash
# 机器 A
cat ~/.chat-room/config.json

# 复制到机器 B
mkdir -p ~/.chat-room
cat > ~/.chat-room/config.json <<EOF
{
  "userId": "新生成的UUID",
  "nickname": "机器B的用户",
  "zkAddresses": ["192.168.1.100:2181"],
  "p2pPort": 9001,
  "currentRoomId": null,
  "recentRooms": []
}
EOF
```

### 7.4 故障预防

#### 定期维护

**每日**：
- 检查 ZooKeeper 状态
- 查看应用日志（如有异常）

**每周**：
- 备份历史消息
- 清理不活跃聊天室

**每月**：
- 检查磁盘空间
- 更新软件版本

#### 监控指标

**关键指标**：

| 指标 | 正常范围 | 异常阈值 |
|------|---------|---------|
| 内存占用 | < 100MB | > 200MB |
| CPU 占用 | < 5% | > 20% |
| P2P 连接数 | 在线成员数 | 持续增长 |
| 消息延迟 | < 500ms | > 2s |

### 7.5 安全建议

#### 网络安全

- 仅在可信的局域网环境使用
- 不要在公共网络暴露 ZooKeeper 端口
- 定期更新依赖包

#### 数据安全

- 历史消息明文存储，注意敏感信息
- 定期清理过期消息
- 备份重要聊天记录

#### 账户安全

- 不要共享配置文件
- userId 是唯一标识，妥善保管
- 退出程序时确保正确关闭

| `Esc` | 取消当前操作 / 退出模式 |
| `@` | 触发成员补全 |
