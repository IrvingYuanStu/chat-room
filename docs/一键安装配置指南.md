# 一键安装配置指南

## 概述

本指南说明如何配置和使用 `curl -fsSL` 方式安装 Chat Room 应用。

## 文件说明

### 1. `install.sh` - 完整安装脚本

功能：
- 检测操作系统
- 自动安装 Node.js（如需要）
- 自动安装 ZooKeeper（如需要）
- 克隆或更新代码仓库
- 安装依赖并编译
- 配置全局命令

使用：
```bash
curl -fsSL https://raw.githubusercontent.com/yourusername/chat-room/main/install.sh | bash
```

### 2. `quick-install.sh` - 快速安装脚本

功能：
- 检查前置条件（Node.js, npm, git）
- 克隆代码仓库
- 安装依赖并编译
- 创建全局命令（可选）

使用：
```bash
curl -fsSL https://raw.githubusercontent.com/yourusername/chat-room/main/quick-install.sh | bash
```

### 3. Docker 部署

文件：
- `Dockerfile` - 容器镜像定义
- `docker-compose.yml` - 多容器编排（包含 ZooKeeper）
- `docker-entrypoint.sh` - 容器启动脚本

使用：
```bash
docker-compose up -d
```

## 配置步骤

### 1. 准备 GitHub 仓库

确保安装脚本可以公开访问：

```bash
# 在仓库根目录
chmod +x install.sh
chmod +x quick-install.sh
chmod +x scripts/*.sh

git add .
git commit -m "Add installation scripts"
git push
```

### 2. 获取原始文件 URL

GitHub Raw 文件 URL 格式：
```
https://raw.githubusercontent.com/[username]/[repo]/[branch]/[filename]
```

示例：
```
https://raw.githubusercontent.com/yourusername/chat-room/main/install.sh
```

### 3. 更新 install.sh 中的下载 URL

编辑 `install.sh`，找到 TODO 注释部分：

```bash
# 下载并提取 chat-room
echo "Downloading chat-room..."
# TODO: 替换为实际的下载 URL
git clone https://github.com/yourusername/chat-room.git "$INSTALL_DIR/chat-room"
```

### 4. 测试安装脚本

在本地测试：

```bash
# 测试完整安装
bash install.sh

# 测试快速安装
bash quick-install.sh
```

### 5. 测试远程安装

从远程仓库测试：

```bash
curl -fsSL https://raw.githubusercontent.com/yourusername/chat-room/main/install.sh | bash
```


## 安全考虑

### 1. 脚本签名

为安装脚本添加 GPG 签名：

```bash
# 生成签名
gpg --detach-sign --armor install.sh

# 验证签名
curl -fsSL https://.../install.sh | gpg --verify
```

### 2. 校验和

提供 SHA256 校验和：

```bash
# 生成校验和
shasum -a 256 install.sh > install.sh.sha256

# 验证校验和
curl -fsSL https://.../install.sh | shasum -a 256 -c install.sh.sha256
```

### 3. 用户确认

在脚本中添加用户确认步骤：

```bash
read -p "This will install Chat Room to $HOME/chat-room. Continue? (y/n) " -n 1 -r
echo
if [[ ! $REPLY =~ ^[Yy]$ ]]; then
    exit 1
fi
```

## 监控和日志

### 安装成功追踪

可以在脚本中添加：

```bash
# 上报安装统计
curl -s https://your-api.com/install/success \
  -d "version=$VERSION" \
  -d "os=$OS" \
  > /dev/null
```

### 错误日志

记录安装失败：

```bash
# 安装失败时上报
trap 'curl -s https://your-api.com/install/failure -d "error=$?"' ERR
```

## 更新和维护

### 1. 自动更新

在应用中添加版本检查：

```typescript
// 检查更新
const checkUpdate = async () => {
  const response = await fetch('https://api.github.com/repos/yourusername/chat-room/releases/latest');
  const latest = await response.json();
  const current = require('./package.json').version;
  if (latest.tag_name !== `v${current}`) {
    console.log(`New version available: ${latest.tag_name}`);
  }
};
```

### 2. 更新通知

在应用启动时提示更新：

```bash
if [ -f "$HOME/chat-room/.update-available" ]; then
    echo "New version available! Run: chat-room-update"
fi
```

## 常见问题

### Q: curl 命令执行权限问题

A: 确保 install.sh 有执行权限，或使用：
```bash
curl -fsSL https://.../install.sh | bash
```

### Q: ZooKeeper 安装失败

A: 参考手动安装 ZooKeeper 的文档，或跳过自动安装。

### Q: 端口冲突

A: 安装后修改配置文件中的端口。

### Q: 如何卸载

A: 删除安装目录和配置：
```bash
rm -rf ~/chat-room ~/.chat-room
```
