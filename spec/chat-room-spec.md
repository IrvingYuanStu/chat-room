## 1. 产品概述

### 1.1 产品定位

Chat Room 是一个运行在 macOS 终端的局域网聊天室工具。它采用 P2P 架构，通过 ZooKeeper 进行节点发现，实现无需中心服务器的去中心化群聊功能。

### 1.2 目标用户

- 需要在局域网内快速搭建临时聊天场景的开发者
- 习惯使用终端进行日常沟通的技术人员
- 需要轻量级沟通工具的小团队

### 1.3 核心价值

- **零配置启动**：无需部署中心服务器，基于 ZooKeeper 自动发现
- **原生终端体验**：完全在终端内操作，符合开发者习惯
- **去中心化**：P2P 通信，无单点故障

## 2. 功能边界

### 2.1 目标 (Goals)

| ID | 目标描述 | 优先级 |
|----|---------|--------|
| G1 | 提供基于终端的实时群聊体验 | P0 |
| G2 | 通过 ZooKeeper 实现自动节点发现与注册 | P0 |
| G3 | 支持 @ 用户和回复消息的互动功能 | P1 |

| NG3 | 消息编辑/删除 | 简化交互复杂度 |

## 3. 用户故事

### 3.1 配置与启动

#### US-001: 首次启动与配置

**作为** 一名新用户
**我想要** 在首次启动时配置 ZooKeeper 地址
**以便** 连接到指定的聊天室网络

**验收标准:**
- [ ] 首次启动时提示用户输入 ZooKeeper 地址
- [ ] 支持输入多个 ZK 地址，以逗号分隔
- [ ] 配置保存到 `~/.chat-room/config.json`
- [ ] 后续启动时显示已保存的配置，允许用户确认或修改

#### US-002: 选择聊天室

**作为** 一名用户
**我想要** 从 ZK 目录中选择或创建聊天室
**以便** 加入目标群聊

**验收标准:**
- [ ] 启动后列出 ZK 目录下所有现有聊天室
- [ ] 支持用户输入新的聊天室 ID（自动创建）
- [ ] 聊天室 ID 为 ZK 目录的最后一段路径

#### US-003: 多聊天室切换

**作为** 一名用户
**我想要** 在不同的聊天室之间切换
**以便** 参与多个讨论

**验收标准:**
- [ ] 支持在多个已加入的聊天室间切换
- [ ] 切换后界面更新为目标聊天室的内容和成员
- [ ] 保持各聊天室的消息历史独立

### 3.2 消息收发

#### US-004: 发送纯文本消息

**作为** 一名用户
**我想要** 在输入框输入文字并发送
**以便** 与其他成员交流

**验收标准:**
- [ ] 左下角输入框显示当前用户昵称
- [ ] 支持单行输入，Enter 发送
- [ ] 支持 Shift+Enter 换行，实现多行输入
- [ ] 发送后消息立即显示在聊天区域，带秒级时间戳

#### US-005: 接收并显示消息

**作为** 一名用户
**我想要** 实时看到其他成员发送的消息
**以便** 参与讨论

**验收标准:**
- [ ] 新消息自动出现在聊天区域底部
- [ ] 消息格式：`[HH:mm:ss] 昵称: 消息内容`
- [ ] 有新消息时终端标题闪烁提示
- [ ] 用户查看历史消息时不自动跳转到底部

#### US-006: @ 用户

**作为** 一名用户
**我想要** 在消息中 @ 其他成员
**以便** 引起特定人的注意

**验收标准:**
- [ ] 输入 `@` 后自动弹出当前在线成员列表
- [ ] 支持键盘/鼠标选择自动补全
- [ ] 被 @ 的消息在聊天记录中高亮显示

#### US-007: 回复消息

**作为** 一名用户
**我想要** 回复某条特定的消息
**以便** 针对性回应

**验收标准:**
- [ ] 支持鼠标选中某条消息后点击"回复"
- [ ] 回复消息显示引用的原始消息内容
- [ ] 回应消息格式：`[HH:mm:ss] 昵称 [回复 原消息]: 回复内容`

### 3.3 用户状态

#### US-008: 上线通知

**作为** 一名用户
**我想要** 知道有新成员加入聊天室
**以便** 了解当前参与人员

**验收标准:**
- [ ] 新成员上线时显示系统消息：`[HH:mm:ss] 系统: 昵称 加入了聊天室`
- [ ] 右侧成员列表实时更新

#### US-009: 下线通知

**作为** 一名用户
**我想要** 知道成员离开聊天室
**以便** 了解当前参与人员变化

**验收标准:**
- [ ] 成员下线时显示系统消息：`[HH:mm:ss] 系统: 昵称 离开了聊天室`
- [ ] 右侧成员列表中该成员状态标记为"离线"

#### US-010: 修改昵称

**作为** 一名用户
**我想要** 修改我的显示昵称
**以便** 使用更友好的名称展示

**验收标准:**
- [ ] 支持通过命令修改昵称（如 `/rename 新昵称`）
- [ ] 修改成功后显示系统消息通知所有成员
- [ ] 新昵称更新到 ZK 节点数据
- [ ] 历史消息中的昵称保持不变，不更新

#### US-011: 查看成员状态

**作为** 一名用户
**我想要** 在右侧看到所有成员及其在线状态
**以便** 了解谁能收到消息

**验收标准:**
- [ ] 右侧成员列表显示所有成员昵称
- [ ] 在线成员显示绿色标识，离线成员显示灰色标识
- [ ] 成员列表按上线时间排序，最新的在前

### 3.4 数据持久化

#### US-012: 消息本地存储

**作为** 一名用户
**我想要** 消息保存在本地
**以便** 下次进入时查看历史记录

**验收标准:**
- [ ] 消息保存到 `/tmp/chat-room/{roomId}/history.txt`
- [ ] 使用纯文本格式，每条消息一行
- [ ] 消息格式：`[HH:mm:ss] 昵称: 消息内容`

#### US-013: 历史消息加载

**作为** 一名用户
**我想要** 进入聊天室时看到历史消息
**以便** 了解之前的讨论内容

**验收标准:**
- [ ] 进入聊天室时加载本地历史文件
- [ ] 最多显示最近 500 条消息
- [ ] 自动滚动到最新消息

#### US-014: 历史消息清理

**作为** 一名系统
**我想要** 定期清理过期历史消息
**以便** 节省存储空间

**验收标准:**
- [ ] 每次启动时清理 1 个月之前的消息
- [ ] 清理操作对用户透明，不显示提示

---

### 3.5 异常处理

#### US-015: ZooKeeper 连接断开

**作为** 一名用户
**我想要** 网络恢复后自动重连
**以便** 继续使用聊天室

**验收标准:**
- [ ] ZK 连接断开时显示警告提示
- [ ] 自动尝试重连，重连间隔 5 秒
- [ ] 重连成功后显示提示消息
- [ ] 重连期间可以继续查看本地历史消息

#### US-016: 成员离线

**作为** 一名用户
**我想要** 给离线成员发送消息时得到提示
**以便** 了解消息可能无法送达

**验收标准:**
- [ ] 给离线成员发送 @ 消息时显示警告
- [ ] 系统不保证离线消息送达

---

### 3.6 退出操作

#### US-017: 退出聊天室

**作为** 一名用户
**我想要** 退出当前聊天室
**以便** 结束会话或切换到其他聊天室

**验收标准:**
- [ ] 输入 `/exit-room` 退出当前聊天室
- [ ] 退出前提示用户确认
- [ ] 退出后返回聊天室选择界面
- [ ] 退出时通知其他成员该用户已离开

#### US-018: 退出程序

**作为** 一名用户
**我想要** 完全退出程序
**以便** 结束聊天

**验收标准:**
- [ ] 支持 `Ctrl+C` 或 `/quit` 退出程序
- [ ] 退出前清理 ZK 临时节点
- [ ] 保存当前配置和未发送消息（如有）

### 3.7 开发测试支持

#### US-019: 本地多客户端测试

**作为** 一名开发者
**我想要** 在同一台机器上启动多个聊天室客户端
**以便** 测试群聊功能而不需要多台机器

**验收标准:**
- [ ] 支持通过命令行参数指定不同的端口号（避免端口冲突）
- [ ] 支持通过命令行参数指定不同的配置文件路径
- [ ] 支持通过命令行参数指定不同的客户端昵称
- [ ] 支持通过命令行参数指定不同的数据存储路径（消息历史、临时文件）
- [ ] 多个客户端实例可以同时连接到同一个 ZooKeeper 服务器
- [ ] 多个客户端实例可以同时加入同一个聊天室
- [ ] 提供启动示例：`chat-room --port 9001 --config ~/.chat-room/client1 --nickname "User1"`

#### US-020: 日志配置与输出

**作为** 一名开发者
**我想要** 将运行日志输出到固定目录
**以便** 调试和问题排查

**验收标准:**
- [ ] 支持通过命令行参数 `--log-dir` 指定日志输出目录
- [ ] 默认日志目录为 `/tmp/chat-room/logs/`
- [ ] 日志文件命名格式：`chat-room-{YYYYMMDD}-{HHMMSS}.log`
- [ ] 支持通过命令行参数 `--log-level` 指定日志级别（debug/info/warn/error）
- [ ] 默认日志级别为 `info`
- [ ] 日志内容包含：时间戳、日志级别、模块名、日志消息
- [ ] 日志格式：`[YYYY-MM-DD HH:mm:ss.SSS] [LEVEL] [MODULE] Message`
- [ ] 提供启动示例：`chat-room --log-dir ~/chat-room-logs --log-level debug`
- [ ] 日志文件支持自动轮转，单个文件最大 10MB

## 5. 数据模型

### 5.1 配置文件结构

**路径:** `~/.chat-room/config.json`

```json
{
  "zkAddresses": ["127.0.0.1:2181", "127.0.0.1:2182"],
  "currentRoomId": "general",
  "nickname": "User001",
  "recentRooms": ["general", "dev-team", "random"]
}
```

### 5.2 ZooKeeper 节点结构

```
/libra-regions
  /{roomId}
    /members
      /{userId} -> {"nickname": "张三", "status": "online", "ip": "192.168.1.100", "port": 9001}
```

### 5.3 消息文件格式

**路径:** `/tmp/chat-room/{roomId}/history.txt`

```
[14:30:25] 系统: 张三 加入了聊天室
[14:30:28] 张三: 大家好
[14:30:35] 李四: @张三 欢迎！
[14:30:40] 张三 [回复 李四]: 谢谢！
[14:31:00] 系统: 李四 离开了聊天室
```

### 5.4 命令行参数

```bash
chat-room [options]

Options:
  --zk-addresses <addresses>    ZooKeeper 服务器地址，多个地址用逗号分隔（默认：127.0.0.1:2181）
  --port <port>                 P2P 监听端口（默认：9001）
  --config <path>               配置文件路径（默认：~/.chat-room/config.json）
  --nickname <name>             客户端昵称（默认：User{随机数}）
  --data-dir <path>             数据存储目录（默认：/tmp/chat-room）
  --log-dir <path>              日志输出目录（默认：/tmp/chat-room/logs）
  --log-level <level>           日志级别：debug/info/warn/error（默认：info）
  --help                        显示帮助信息
```

### 5.5 日志文件格式

**路径:** `{log-dir}/chat-room-{YYYYMMDD}-{HHMMSS}.log`

```
[2026-04-02 14:30:25.123] [INFO] [App] 应用启动，版本 1.0.0
[2026-04-02 14:30:25.456] [INFO] [Config] 加载配置文件: /Users/user/.chat-room/config.json
[2026-04-02 14:30:26.789] [INFO] [ZKClient] 连接 ZooKeeper: 127.0.0.1:2181
[2026-04-02 14:30:27.012] [DEBUG] [P2PServer] 启动 P2P 服务器，端口: 9001
[2026-04-02 14:30:27.345] [INFO] [Room] 加入聊天室: general
[2026-04-02 14:30:30.678] [WARN] [Network] 检测到网络波动，尝试重连...
[2026-04-02 14:30:31.901] [ERROR] [Peer] 无法连接到节点 192.168.1.100:9002
```

## 6. MVP 里程碑规划

### Milestone 1: 基础框架 (Week 1)

**目标:** 可运行的终端 UI 和 ZK 连接

**范围:**
- 终端 UI 布局实现（三栏结构）
- ZK 连接与节点注册
- 配置文件读写
- 聊天室选择界面
- 命令行参数支持
- 日志系统实现

**验收标准:**
- 能够启动程序并配置 ZK 地址
- 能够看到并选择聊天室
- 能够通过命令行参数配置端口、昵称、数据目录
- 能够通过命令行参数配置日志输出
- 日志正常输出到指定目录

### Milestone 2: 消息收发 (Week 2)

**目标:** 基本的 P2P 消息收发

**范围:**
- TCP P2P 连接建立
- 文本消息发送与接收
- 消息显示与时间戳
- 成员上线/下线通知

**验收标准:**
- 两个终端能够互相收发消息
- 能够看到成员变化通知

### Milestone 3: 互动功能 (Week 3)

**目标:** @ 和回复功能

**范围:**
- @ 自动补全
- 鼠标选中回复
- 消息高亮显示
- 昵称修改

**验收标准:**
- 能够 @ 其他用户并看到高亮
- 能够回复特定消息

### Milestone 4: 数据与异常 (Week 4)

**目标:** 完整的数据持久化和异常处理

**范围:**
- 消息本地存储与加载
- 历史消息清理
- ZK 断线重连
- 多聊天室切换

**验收标准:**
- 重启后能看到历史消息
- 网络波动后自动恢复

## 7. 非功能需求

| ID | 需求描述 | 指标/要求 |
|----|---------|----------|
| NFR-1 | 消息延迟 | < 500ms (局域网环境) |
| NFR-2 | 同时在线人数 | 支持 50+ 成员 |
| NFR-3 | 内存占用 | < 100MB |
| NFR-4 | 启动时间 | < 3 秒 |
| NFR-5 | 消息历史 | 最多 500 条 |
| NFR-6 | 本地多客户端支持 | 单机支持至少 5 个并发客户端实例 |
| NFR-7 | 日志性能 | 日志写入不影响消息收发性能 |
| NFR-8 | 日志文件大小 | 单个日志文件最大 10MB，自动轮转 |

## 8. 术语表

| 术语 | 定义 |
|-----|------|
| ZK | ZooKeeper，用于节点注册和发现的协调服务 |
| P2P | Peer-to-Peer，点对点通信模式 |
| 聊天室 ID | ZK 目录路径的最后一段，用于标识唯一聊天室 |
| 临时节点 | ZK 的一种节点类型，客户端断开后自动删除 |
| TCP | 传输控制协议，用于 P2P 消息传输 |
